ar5iv VS Code

A VS Code editing surface for LaTeX documents that reuses the same ar5iv conversion, preview rendering, and source-map model as the web editor — sharing one frontend-core/ across all three surfaces.

Browser VS Code (this route)

/vscode serves a self-hosted VS Code for the Web workbench with the ar5iv extension preloaded as a built-in and conversions pointed at this server. You are seeing this launcher page because the VS Code Web build has not been vendored yet. To enable the workbench: 

cd vscode-extension
npm install
npm run fetch:vscode-web   # downloads the pinned VS Code Web build into ./vscode-web
npm run build              # bundles the extension + webview assets

Then restart the server (it serves ./vscode-web at /vscode-static and the extension at /vscode-ext), and reload this page. Override the build location with AR5IV_VSCODE_WEB_DIR.

Local Extension (the “code” plugin)

The same extension in vscode-extension/ runs as a desktop VS Code plugin: a persistent preview webview that refreshes on edits, publishes diagnostics, and supports two-way source-map navigation. Build it with npm run build:all (or npm run package:vsix) and launch it from VS Code's extension development host. Desktop auto mode runs a managed local ar5iv-editor server; set ar5iv.conversionMode/ar5iv.backendUrl for explicit hosted testing.
