Schema reference

Browseable RelaxNG schemas for the formats this editor emits and validates against.

Each entry below is a generated documentation site — module overviews, element / attribute / pattern definitions, content models, and “used by” cross-references — built fresh from the source .rnc files at image-build time. The pipeline is trang → genschema_oxide → latexml_oxide --schemadocs.

Schema What it describes Source
LaTeXML.rnc The native intermediate XML produced by the LaTeXML engine before any HTML / ePub / JATS post-processing. Defines ltx:document, ltx:section, ltx:Math, and the rest of the abstract document vocabulary that drives every downstream profile. latexml-oxide
scholarly-ltx.rnc The scholarly HTML5 dialect emitted for arXiv-style papers — standard HTML5 plus the ltx_* class vocabulary, MathML 4 (Core) for math rendering, and the structural conventions that survive the ltx → HTML transformation. Content MathML is permitted for backwards compatibility with older corpora; new content should target Core. The same schema the W3C Nu validator uses when invoked with the “HTML scholarly” preset. validator
mathml4-core.rnc A minimal, browser-implementable subset of MathML 4 covering only the constructs every modern engine (Chromium-derived, WebKit, Gecko) renders today. Useful when you want to guarantee a fragment displays consistently in browsers without falling back to presentation-only, image, or polyfill paths. mathml-schema

How to read these

Each generated site groups definitions per source module (LaTeXML-block.rnc, scholarly-ltx-math.rnc, …) and uses a rustdoc-styled theme: the left rail navigates modules, each definition carries a stable #schema.name anchor with a § permalink on hover, and content models render as multi-line structural expressions rather than one-line walls of comma-separated links.

Validating documents against these schemas

All three schemas are live validation targets, not just reference documentation. POST a document to /api/validate and the service — a hosted Nu validator carrying these schemas as built-in presets — returns the full report. The request Content-Type selects the schema:

Content-Type Validates against
text/html LaTeXML scholarly HTML5 profile (scholarly-ltx.rnc) — also the default when no Content-Type is sent.
application/latexml+xml LaTeXML XML document schema (LaTeXML.rnc). The generic application/xml and text/xml are accepted as aliases.
application/mathml+xml MathML 4 Core (mathml4-core.rnc).

Reports default to the Nu validator's JSON shape ({"messages": […]}; an empty list means the document is valid); pass ?out=gnu, ?out=xml, or ?out=text for other formats. Documents must be UTF-8 and at most 16 MB; unsupported media types get a 415 listing the supported set. 

# scholarly HTML5 (the default)
curl -X POST https://latexml.rs/api/validate \
     -H 'Content-Type: text/html; charset=utf-8' \
     --data-binary @paper.html

# the engine's native XML
curl -X POST https://latexml.rs/api/validate \
     -H 'Content-Type: application/latexml+xml' \
     --data-binary @paper.xml

# a MathML Core fragment, as a GNU-style error listing
curl -X POST 'https://latexml.rs/api/validate?out=gnu' \
     -H 'Content-Type: application/mathml+xml' \
     --data-binary @formula.mml
